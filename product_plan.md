# 용돈/재테크 관리 앱 제안서 (나 + 아이들)

## 1) 핵심 요구사항 정리
- 사람(가족 구성원) 단위 관리: `나`, `첫째`, `둘째`로 시작하고 이후 자유롭게 추가
- 일자별 용돈 수입 기록 및 기간별 합계
- 현재 자산 배분 확인
  - 저축(예금/적금/저금) 금액
  - 투자 자산 금액
  - 비율(%) 시각화
- 투자 방식/상품별 기록
- 재테크 유형별 수익률(기간/누적) 계산
- 대시보드(메인)에서 한눈에 요약
- 데이터 영속 저장(기기 변경/재설치 후에도 유지)
- 어디서든 접속(웹/모바일)
- 계정 기반 로그인(구글 아이디)

## 2) 추천 MVP 기능
1. **인증/권한**
   - Google 로그인
   - 가족(그룹) 생성/초대
2. **사람 관리**
   - 기본 멤버(나/첫째/둘째) 생성
   - 멤버 추가/수정/비활성
3. **거래 기록**
   - 수입(용돈), 저축, 투자, 지출(선택) 공통 입력 폼
   - 날짜/금액/대상(사람)/메모
4. **자산 현황**
   - 저축 vs 투자 비율
   - 사람별 현재 총자산
5. **성과 분석**
   - 재테크 카테고리별 수익률
   - 월별 누적 그래프
6. **대시보드**
   - 오늘/이번달 수입
   - 사람별 자산 카드
   - 비율 도넛차트 + 최근 활동

## 3) 데이터 모델(초안)
- `users`: 로그인 계정
- `families`: 가족 단위(워크스페이스)
- `members`: 관리 대상 사람(나/첫째/둘째...)
- `accounts`: 저축/투자 계좌(예금, 적금, ETF, 주식 등)
- `transactions`: 모든 입출금/이동 기록
- `positions`(선택): 투자 보유 수량/평단
- `daily_snapshots`(선택): 일별 평가금액 캐시(대시보드 속도용)

## 4) 수익률 계산 방식 제안
- MVP: 단순 수익률
  - `(현재평가금액 - 원금) / 원금 * 100`
- 고급(2차): 현금흐름 반영 수익률
  - MWR(내부수익률), TWR 지원
- 화면에서는 기본 수익률 + 기준일(예: 2026-04-05 기준) 명시

## 5) 기술 스택 추천(어디서든 접속 + iPhone 배포 고려)
### 옵션 A (추천): **Flutter + Firebase**
- 장점
  - iOS/Android/Web 동시 지원
  - Google 로그인, DB, 호스팅까지 빠른 구현
  - 1인/소규모 MVP 개발 속도 우수
- 구성
  - 앱: Flutter
  - 인증: Firebase Authentication (Google)
  - 데이터: Cloud Firestore
  - 파일/백업: Firebase Storage (필요 시)
  - 웹 배포: Firebase Hosting

### 옵션 B: **React(Next.js) + Supabase + React Native(Expo)**
- 웹 우선이면 유리
- 모바일 앱 추가 시 구성 복잡도는 옵션 A보다 다소 높음

## 6) iPhone 앱 배포 가능 여부
**가능합니다.**
- Apple Developer 계정 필요(유료)
- 빌드/테스트(TestFlight) 후 App Store 배포
- Google 로그인은 iOS에서도 정상 지원
- 심사 시 아동 관련 정책(연령, 개인정보) 문구를 명확히 준비해야 함

## 7) 보안/운영 체크리스트
- 가족 단위 접근 제어(내 가족 데이터만 조회)
- 전송/저장 암호화(플랫폼 기본 + 규칙)
- 변경 이력(누가 언제 수정했는지)
- 정기 백업/내보내기(CSV)
- 개인정보처리방침/서비스 이용약관 준비

## 8) 단계별 개발 로드맵 (예시)
- 1주차: 인증 + 가족/멤버 생성 + 거래 입력
- 2주차: 대시보드(합계/비율/사람별 카드)
- 3주차: 수익률 계산 + 필터(기간/사람/카테고리)
- 4주차: 웹 배포 + iOS 빌드(TestFlight)
- 5주차: 피드백 반영 후 App Store 심사 제출

## 9) 화면 구성 제안
- `대시보드`: 전체 합계, 저축/투자 비율, 최근 기록
- `거래 입력`: +버튼으로 빠른 등록
- `분석`: 재테크별 수익률/기간 비교
- `멤버`: 사람 추가/관리
- `설정`: 로그인 계정, 데이터 내보내기, 권한

## 10) 바로 시작할 때 필요한 결정 5가지
1. 통화 단위(원 고정인지)
2. 투자 유형 분류(주식/ETF/코인/펀드 등)
3. 수익률 기준(단순/현금흐름 반영)
4. 가족 내 권한(조회 전용/편집 가능)
5. 알림 필요 여부(정기 입력 리마인드)
